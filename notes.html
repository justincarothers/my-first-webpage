Notes on Day 1 of Epicodus

Table of Command Line Commands

Command Name arguments An explanation of what the command does with those arguments
Navigating Directories
pwd Stands for "print working directory." Displays the file path of the current directory
ls Shows list of items in current directory
Editing Files
touch file name creates new file with "file name"
mkdir directory name creates a new directory with the name "directory name"
Git commands
git init Sets up a local git repository in the current working directory
git pair two sets of initials corresponding to the initials in the pair document in the parent directory establishes the pair of people who will be working with git for the day
git add filename.html adds a file (in the same directory the repository is in) to the list of files git will keep track of when it commits, pushes, and clones
git-pair-commit -m creates a git commit using the pair previously established using git pair
git remote add one pair of initials, github clone url links the local git repository to the git repository in the cloud - allows for subsequent pushing and cloning to and from github.com
git status Displays whether the git repository is up to date with the changes being made to the files in the directory. Prompts user to use git add if not.
git about Displays names of collaborators, email addresses of collaborators, and project title if there is one.

Table of HTML Tags

td create column
body creates body

Setting up the local git repository and making the first local commit

  • git init
  • git pair first and second initials
  • IF PAIRED: git-pair-commit -m "description of the changes in this commit"
  • IF SOLO: git commit -m "description of the cnages in this commit"
